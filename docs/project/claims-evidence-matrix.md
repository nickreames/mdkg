# Claims Evidence Matrix

Public mdkg.dev copy should map claims to shipped behavior and caveats.

Unsupported public claims must stay blocked until the corresponding behavior is implemented, validated, and intentionally launched.

Canonical source for the current site scaffold:

```text
mdkg-dev/CLAIMS.md
```

The matrix must include page, claim, evidence source, shipped status, caveat or safe wording, owner, and review status. Use it before changing homepage, docs, social metadata, or launch copy.

Do not claim worker execution, hosted memory, hosted queues, arbitrary SQL, comprehensive secret scanning, broad MCP mutation parity, universal agent compatibility, or production mdkg.dev launch unless those surfaces are separately implemented and verified.
