Advanced alpha surfaces are useful for larger agentic workflows, but they are not the first-run path.

- [Project DB and queues](/advanced-alpha/project-db-queues/)
- [Read-only MCP](/advanced-alpha/read-only-mcp/)
- [Subgraphs and bundles](/advanced-alpha/subgraphs-and-bundles/)
- [Graph movement](/advanced-alpha/graph-movement/)
- [Demo graphs](/advanced-alpha/demo-graphs/)
- Archives and workflow mirrors

Each advanced feature should be documented with what it is, when to use it, when not to use it, safety boundaries, example commands, current limitations, and stability caveats.

Use these surfaces only after `mdkg init --agent`, `mdkg index`, `mdkg validate`, and the basic Plan -> Work -> Evidence loop are comfortable in a small repository.
