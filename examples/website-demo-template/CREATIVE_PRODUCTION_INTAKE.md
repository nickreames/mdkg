# Creative Production Intake

Use this contract when a demo run wants Creative Production ideation before
implementation. Creative Production is optional; the demo must still be
buildable from `goal-1`, `spike-1`, and the files in this template.

## Role

Creative Production helps choose the creative direction. It may propose
positioning, visual territories, page structure, section rhythm, motion ideas,
interaction ideas, imagery, and asset needs. It does not own final website code,
Vercel deployment, DNS, analytics, or public release claims.

## Minimum Brief

Before asking for creative options, provide these facts from the current demo
run:

- what the website is for;
- who the intended audience is;
- what the visitor should understand or do;
- any market, channel, or context that matters;
- content, product, or feature facts that must stay source-backed;
- available public-safe assets or references;
- what must appear;
- what to avoid;
- whether the output should become a landing page, docs-like experience,
  product narrative, interactive demo, campaign page, or another website shape.

If any item is unknown, mark it unknown instead of inventing hidden context.

## Suggested Explore Ask

Use Creative Production Explore when available. Keep the ask compact and bounded
to public-safe website ideation:

```text
Use Creative Production Explore for this mdkg website demo brief. Preserve the
Ocean Flow design system, Astro plus React Islands stack, source-backed mdkg
claims, accessibility, and no-secret boundaries. Help choose the strongest
website direction, including audience, offer, page structure, visual territory,
motion ideas, interaction ideas, and asset needs. Do not generate or retain raw
private prompts, provider payloads, credentials, or deployment material.
```

Useful Explore paths for this template are usually Positioning, Mood boards,
Offers, Scenes, Ads, or Shots. Use Assets only after a direction is selected and
the run intentionally needs durable public-safe images or files.

## Creative Freedom

Creative Production may vary:

- audience angle and visitor promise;
- page structure and section order;
- hero concept;
- art direction within Ocean Flow;
- motion and animation style;
- React Island interaction concepts;
- generated or sourced public-safe assets;
- tone, rhythm, and supporting microcopy.

Creative Production may not vary:

- Astro plus React Islands as the implementation stack;
- Ocean Flow as the baseline visual system;
- accessibility and responsive requirements;
- source-backed mdkg claims;
- no-secret, no-raw-payload, and no-private-context rules;
- local-first validation before preview approval;
- the ban on unapproved deploy, push, publish, DNS, analytics, durable hosting,
  or npm release work.

## Direction Summary

After ideation, record a compact summary in `spike-1` before starting `task-1`.
Do not paste raw prompt transcripts or provider payloads. Include:

- audience;
- core promise or offer;
- page structure;
- visual territory;
- motion and interaction ideas;
- React Island candidates;
- asset plan;
- source-backed content facts;
- explicit non-goals;
- risks to check in `test-1`.

The coding agent should be able to implement the website from that summary,
`DESIGN.md`, `WEBSITE_DEMO_TEMPLATE_BRIEF.md`, `dec-1`, `dec-2`, and `edd-1`
without hidden chat context.
