# Context

The audit asks for a more polished OSS landing-page feel and one static Plan -> Work -> Evidence diagram, but the immediate goal is launch-quality product clarity, not a new asset-production program.

# Decision

Creative Production is used in goal-34 as a review and direction gate for visual hierarchy, diagram clarity, and copy polish. Generated image/video assets remain follow-up work unless explicitly rescoped.

# Alternatives considered

- Generate full visual asset packs during pass 4.
- Keep pass 4 CSS/HTML first, using Creative Production for critique and direction.

# Consequences

- The pass stays focused on concrete UX/docs quality while still benefiting from creative review.
- Future generated imagery, demo videos, or launch assets can be planned as separate mdkg work.
