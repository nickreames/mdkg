# Decision

Keep the Claims Evidence Matrix as internal quality-control evidence. Do not expose it in public docs navigation unless it is rewritten as a polished public trust page.

# Rationale

The public user needs safety boundaries, trust posture, and accurate claims. A raw internal matrix reads like governance scaffolding and makes the docs feel unfinished.

# Consequences

- Public navigation should prefer Safety Boundaries and mdkg.dev Trust.
- If a matrix page remains deployed, it must be hidden from nav and marked noindex or moved out of the public docs route.
- Goal 33 should verify the public nav does not surface internal evidence pages.
