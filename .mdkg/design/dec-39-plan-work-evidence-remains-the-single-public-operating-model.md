# Decision

Use `Plan -> Work -> Evidence` as the only public mdkg operating model across mdkg.dev, docs.mdkg.dev, README-aligned copy, and future public examples.

# Rationale

The phrase is concrete, memorable, and maps directly to mdkg graph behavior: planning nodes, bounded work nodes, evidence/checkpoints, and validation. Competing public phrases dilute comprehension.

# Consequences

- Remove or avoid `golden loop`, `repo memory loop`, and generic `operating loop` language in public-facing copy.
- Preserve implementation details for maintainer-facing docs when needed.
- Use setup language separately from the ongoing Plan -> Work -> Evidence loop.
