# Goal

Dogfood the new spike type against mdkg.dev launch preparation so the feature
proves useful for real research, documentation planning, skill candidates, and
follow-up node creation.

# Scope

- Use spikes to structure mdkg.dev launch research across data structures,
  algorithms, UX, security, docs, SEO, and marketing readiness.
- Create follow-up mdkg nodes and skill-authoring candidates from spike
  recommendations.
- Keep website implementation deferred until generated docs and launch goals are
  ready.

# Milestones

- Spike template includes mdkg.dev-oriented research output sections.
- Packed smoke proves a spike can produce follow-up work and skill candidates.
- Dogfood task records the first mdkg.dev launch research backlog.

# Out of Scope

- No mdkg.dev website build in this goal.
- No public worker execution or public internal-DB command surfaces.
- No downstream repo migration automation.

# Risks

- Launch research can become vague unless every spike ends in explicit
  recommendations and follow-up nodes.
- SEO/marketing work should wait until command docs and capability narratives
  are stable enough to publish.
