# Overview

Enhance mdkg-authored `SKILL.md` workflows for research-backed docs and site
work after spike dogfooding exposes repeatable patterns. Skills should capture
durable operator behavior without claiming mdkg itself executes research.

# Acceptance Criteria

- Identify repeated spike-to-docs workflows worth turning into skill guidance.
- Update or create skills for research-backed documentation, public examples,
  docs verification, and launch closeout where justified.
- Sync mirrored skills after canonical `.mdkg/skills` edits.
- Validate skills and graph state after changes.

# Files Affected

- `.mdkg/skills/`
- `.agents/skills/`
- `.claude/skills/`
- related mdkg work nodes

# Implementation Notes

- Use `author-mdkg-skill` when authoring or changing skill files.
- Keep `.mdkg/skills` canonical and mirrors generated by `mdkg skill sync`.
- Do not add skills for one-off tasks that have not repeated.

# Test Plan

- `mdkg skill validate <slug>` for changed skills.
- `mdkg skill sync` when mirrors need refresh.
- `node dist/cli.js validate --json`.
