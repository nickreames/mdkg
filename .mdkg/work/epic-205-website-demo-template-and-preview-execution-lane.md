# Goal

Create the executable lane for `goal-44`: canonical website demo template,
branch-path run proof, local validation, accepted Demo 1 source integration in
mdkg-dev at `/demos`, `/demo/1`, and `/demo/1/output`, and closeout
recommendation.

# Scope

- Canonical template path: `examples/website-demo-template/`
- First run path: `examples/demo-runs/demo-001/`
- Public accepted-demo paths: `/demos`, `/demo/1`, `/demo/2`, etc.
- Goal-only startup, pack, local Browser/Chrome checks, no-secret/public-claims
  audit, route evidence contracts, and homepage isolation proof.

# Milestones

- `task-618` creates the template graph and handoff.
- `task-620` proves branch-path fork/run mechanics.
- `task-621` is historical/superseded local preflight for the earlier preview
  project model.
- `task-628` plans mdkg-dev short-path Astro route integration.
- `task-629` creates the sanitized Demo 1 graph/filesystem/output snapshot.
- `task-630` implements the local demo index, detail, and output routes.
- `task-622` records accept/reject/polish recommendation.

# Out of Scope

- Dedicated demo Vercel project creation, DNS, custom domains, aliases, durable
  `demo-N.mdkg.dev` hosting, push, tag, npm publish, production promotion, and
  analytics activation.

# Risks

- Demo output overclaims mdkg capabilities.
- Creative artifacts retain private context.
- Short-path demo output is published before snapshot safety is proven.
- Viewer code harms homepage or docs loading.
- Fork/run path is not repeatable.
