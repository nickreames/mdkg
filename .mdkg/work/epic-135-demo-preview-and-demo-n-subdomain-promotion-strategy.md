# Goal

Plan how future demo previews can become curated `demo-N.mdkg.dev` subdomains without destabilizing canonical mdkg.dev.

# Scope

- Preview-first demo validation.
- Durable demo subdomain promotion criteria.
- Teardown and rejection policy for throwaway demos.
- Separation from canonical SEO surface.

# Milestones

- Define when a demo remains a preview URL.
- Define when a demo can be promoted to a durable subdomain.
- Define evidence needed for teardown/promotion decisions.

# Out of Scope

- Creating demo deployments.
- Assigning demo subdomains.
- Changing canonical mdkg.dev content strategy.

# Risks

- Demo sites may create inconsistent public claims if promoted too early.
- Search engines may index demos unless noindex/canonical policy is explicit.
